HASIL ANALISIS API ANIMEINWEB.COM (UPDATE)
========================================

1. STRUKTUR DASAR API
--------------------
Website menggunakan Proxy API untuk komunikasi backend.
Base URL Proxy: https://animeinweb.com/api/proxy

2. AUTENTIKASI (User Identity)
------------------------------
Data identitas user dikirim sebagai parameter query atau form data:
- id_user: [ID User dari localStorage auth_pref_v1]
- key_client: [Key Client dari localStorage auth_pref_v1]

3. ENDPOINT KONTEN ANIME
-------------------------
Semua endpoint di bawah ini diakses melalui Base Proxy (Contoh: BaseURL + /3/2/home/data)

A. Home Data (Jadwal & Update)
   Endpoint: /3/2/home/data
   Query Params: day (SENIN, SELASA, dll), limit

B. Explore & Search (Pencarian)
   Endpoint: /3/2/explore/movie
   Query Params: 
     - page: [number]
     - sort: [new, popular, old, stars]
     - keyword: [string] (untuk pencarian judul)
     - genre_in: [id_genre] (filter genre)

C. Genre List
   Endpoint: /3/2/explore/genre
   Kegunaan: Mengambil daftar semua genre dan ID-nya.

D. Detail Anime
   Endpoint: /3/2/movie/detail/:id
   Kegunaan: Mengambil sinopsis, skor, info studio, dan status anime.

E. Daftar Episode
   Endpoint: /3/2/movie/episode/:id
   Query Params: page, search

F. Streaming & Link Server
   Endpoint: /3/2/episode/streamnew/:id
   Kegunaan: Mengambil link video (server), subtitle, dan info episode selanjutnya.

4. ENDPOINT INTERAKSI & CHAT
-----------------------------

A. Chat (Polling)
   - Fetch: /3/2/chat/data (Params: highest_id, id_user, key_client)
   - Send Text: /3/2/chat/do (Body URLSearchParams: text, id_chat_replay, id_user, key_client)
   - Send Image: /3/2/chat/do (Body Multipart/Form-Data:
        - text: [string, untuk caption/teks gambar]
        - id_chat_replay: [number]
        - id_user: [ID User]
        - key_client: [Key Client]
        - image: [File buffer gambar berformat .jpg atau .png. Format .jpeg ditolak server])

B. Komentar (Episode)
   - Data: /3/2/comment/data (Params: id_episode, sort, page)
   - Post: /3/2/comment/do (Body: text, id_episode)
   - Action: /3/2/comment/action (Body: id_comment, type [Y/N])

5. ENDPOINT PENGGUNA (USER)
---------------------------
- Login: /auth/login
- Register: /auth/register
- Stream Log: /3/2/user/stream/do (Untuk mencatat progres tontonan)

7. MEDIA DOMAIN
---------------
- Images: https://xyz-api.animein.net (Prefix untuk path gambar)

8. METADATA JSON KEYS (Hasil Analisis Browser)
-----------------------------------------------
Berdasarkan inspeksi langsung pada endpoint /explore/movie dan /movie/detail:

A. Keys Utama Anime:
   - "title": Judul utama
   - "synonyms": Judul alternatif (Alt Titles)
   - "views": Jumlah penonton (tipe string angka)
   - "favorites": Peringkat/Suka (digunakan sebagai indikator Rating di UI)
   - "aired_start": Tanggal tayang pertama (YYYY-MM-DD)
   - "year": Tahun tayang (angka saja)
   - "synopsis": Deskripsi/Sinopsis cerita
   - "studio": Nama studio (biasanya ada di endpoint Detail)
   - "genre": Daftar genre (comma separated string)
   - "image_poster": URL path poster
   - "image_cover": URL path cover/banner

B. Struktur Detail Endpoint (/movie/detail/:id):
   - Data berada di dalam: res.data.data.movie
   - Berisi informasi lebih mendalam termasuk studio dan daftar genre lengkap.

C. Struktur Explore Endpoint (/explore/movie):
   - Data berada di dalam: res.data.data.movie (Array)
   - Berisi list anime dengan informasi dasar termasuk views, favorites, dan aired_start.
